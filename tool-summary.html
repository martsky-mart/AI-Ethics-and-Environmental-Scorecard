Methodology & Scope of the Assessment

The acceleration of Generative AI tools in digital creative pipelines and code development workspaces has created a dual challenge: addressing the Scope 3 carbon footprint associated with computing-intensive model inference, and managing the intellectual property liability that accompanies deep-learning training architectures.

This report compiles detailed dataset metadata from the AI Tool Breakdown spreadsheet. It tracks which specific external foundational models are leveraged behind the scenes for each piece of software Hive employs. Rather than treating each tool as a monolithic entity, we evaluate the system at the model provider level to discover downstream vulnerabilities.

Action Plan

Recommended ESG Mitigations

Actionable interventions targeting carbon output and licensing liability within Hive operations.

1

Commercial Safety Mandates

Formally restrict Canva Magic and Figma AI for client-facing deliverables. Enforce Adobe Firefly or Getty Generative AI tools as primary engines, which provide indemnification guarantees against claims.

2

Scope 3 Emissions Targets

Define a workspace emissions ceiling equivalent to 1,000 kg of CO₂e per month. Integrate programmatic alerts within GitHub API pipelines to check if unused testing repositories are triggering automated code generations.

3

Local Model Infrastructure

Encourage developer workstations to deploy local open-weights engines (e.g., Llama 3 via Ollama) rather than routing raw queries to centralized proprietary cloud endpoints when processing non-sensitive datasets.
